# MVP実装計画

## 目的

離島の空き家について、現況の3Dモデルを基準に、単一視点で破綻しにくいリフォーム後イメージを提示できる最小構成を作る。

## 開発方針

- まずは単一視点での品質と再現性を優先する
- 3Dは正解データ、生成AIはその上に乗る提案レイヤーとして扱う
- 視点をまたいだ整合性や完全自動化はMVPの対象外とする

## フェーズ1: Three.jsデータ抽出

- `.glb` 表示
- カメラ操作
- 通常画像のキャプチャ
- Depth表示切り替え
- Depth画像キャプチャ
- カメラJSON保存
- world position取得
- マスク描画とマスクPNG保存

## フェーズ2: 生成入力の整理

- 出力ファイル命名規則の確定
- 1視点分の入出力セット定義
- 保存フォルダ構成の確定
- ComfyUIへ渡す入力仕様の固定

## フェーズ3: ローカル画像生成

- ComfyUI 前提の最小ワークフロー設計
- 入力: color / depth / mask / prompt
- 出力: 単一視点のリフォーム後画像
- seed、モデル名、prompt の保存

## フェーズ4: 比較UI

- Before / After 比較表示
- 同一視点での複数案比較
- 生成条件の表示
- AI生成画像であることの明示
